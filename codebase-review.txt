OVERVIEW:
---------
A single-page portfolio website built with React 19, TypeScript, and Vite.
Uses scroll-based view rendering with Intersection Observer for dynamic 
component loading. Targeting Vercel deployment.

Tech Stack:
- React 19.2.0 with React Compiler (babel-plugin-react-compiler)
- TypeScript 5.9.3
- Vite 7.2.4
- ESLint 9 with flat config
- Vercel Speed Insights

================================================================================
                              CONS / ISSUES
================================================================================

1. MISSING VERCEL CONFIGURATION [CRITICAL]
   ✗ No vercel.json file exists
   ✗ For SPA with client-side routing, needs rewrites configuration
   ✗ Missing CSP (Content Security Policy) headers
   
   Impact: May cause 404 errors on page refresh or direct URL access

2. INCONSISTENT/MISSING INDEX EXPORTS
   ✗ src/components/home-page/index.ts exists but not used in main index.ts
   ✗ about-page, projects-page, contact-page lack index.ts files
   ✗ Inconsistent import patterns across the codebase

3. DUPLICATE SECTION IDS
   ✗ HomePage wraps content in <section id="home"> 
   ✗ ScrollView also receives id="home" - results in duplicate IDs
   ✗ Same issue for about, projects, contact sections
   
   Impact: Invalid HTML, potential accessibility/navigation issues

4. MISSING/EMPTY STYLES
   ✗ about-page.css is empty (0 bytes)
   ✗ No dedicated styles for projects-page and contact-page
   ✗ Placeholder pages (ProjectsPage, ContactPage) lack styling

5. DUPLICATE CSS RULES
   ✗ falling-star.css has duplicate z-index declaration (lines 24-25)
   ✗ app.css has both fadeIn keyframes AND scroll-view.css redefines fadeIn
   
   Impact: Potential style conflicts, increased bundle size

6. ANIMATION TIMING MISMATCH [POTENTIAL BUG]
   ✗ falling-star.tsx: setTimeout removes stars after 15000ms
   ✗ But animation duration is "15 + Math.random() * 50" seconds (15-65s)
   ✗ Stars may be removed before animation completes

7. TYPE DEFINITION GAPS
   ✗ src/types/index.ts only exports hooks.types.ts
   ✗ No component prop types centralizedP
   ✗ ScrollView uses inline interface instead of shared type

8. HARDCODED VALUES
   ✗ nav top position (68px, 66px) duplicates --header-height logic
   ✗ Magic numbers in animations (2s, 2.5s, etc.) not documented
   ✗ Profile picture path duplicated in home-page.tsx and about-page.tsx

9. SEO/META IMPROVEMENTS NEEDED
   ✗ Missing meta description
   ✗ Missing Open Graph tags for social sharing
   ✗ Missing canonical URL
   ✗ Only favicon is a PNG (should also have SVG/ICO variants)

10. BUILD CONFIGURATION
    ✗ gh-pages script exists but conflicts with Vercel deployment
    ✗ No build output optimization configuration in vite.config.ts
